## Analysis & Reasoning
**`prisma:lint` — dotenv version mismatch**

The PR changed `"dotenv": "^16.4.7"` to `"dotenv": "17.4.2"` in `libs/shared/prisma/package.json`.
The `@nx/dependency-checks` ESLint rule validates that the version specifier listed in a project's `package.json` is satisfied by the version actually installed in `node_modules`.
The installed version is `16.4.7`, which does not satisfy the exact specifier `17.4.2`.
Reverting to `"^16.4.7"` (a semver range that includes `16.4.7`) resolves the lint error without requiring an actual dotenv upgrade.

---

**`prisma:build` — TS6059 rootDir violation**

The PR refactored `libs/shared/prisma/src/seed.ts` to import from `@job-tracker-lite-angular/shared-testing`.
The `@nx/js:tsc` executor infers `rootDir` for the prisma project from its source root (`libs/shared/prisma`).
When TypeScript resolves the path alias `@job-tracker-lite-angular/shared-testing` → `libs/shared/testing/src/index.ts`, all transitively imported files from the testing library fall outside this inferred `rootDir`, triggering TS6059.

The decision was made to **exclude `seed.ts` from `tsconfig.lib.json`** rather than widen `rootDir` or introduce TypeScript composite project references, for these reasons:

- `seed.ts` is a standalone database seeding script, not part of the library's public API (it is not exported from `index.ts`).
- The `seed` Nx target already invokes it via `ts-node --project tsconfig.json`, which uses the workspace root `tsconfig.base.json` with `rootDir: "."` — covering the entire monorepo. The seed script therefore compiles and runs correctly without being part of the library build.
- Widening `rootDir` in `tsconfig.lib.json` would alter the output directory structure of compiled declarations.
- Adding TypeScript composite project references would require adding `composite: true` to the testing library and updating multiple tsconfigs — a larger change than necessary.
